Element loader-repository
diagram
description The loader-repository specifies the name of the UnifiedLoaderRepository MBean to use for the ear to provide ear level scoping of classes deployed in the ear. It is a unique JMX ObjectName string. It may also specify an arbitrary configuration by including a loader-repository-config element. Examples: &lt;loader-repository&gt;jboss.test:loader=cts-cmp2v1-sar.ear&lt;/loader-repository&gt; &lt;loader-repository loaderRepositoryClass='dot.com.LoaderRepository'&gt; dot.com:loader=unique-archive-name &lt;loader-repository-config configParserClass='dot.com.LoaderParser'&gt; java2ParentDelegaton=true &lt;/loader-repository-config&gt; &lt;/loader-repository&gt;

uses Does not use other elements.
used by server
content empty

Attribute Datatype Required Values Default Comments
codebase string (default) Yes     The codebase attribute specifies the base URL from which the jars or classes are loaded from. If the codebase is '.' then it is equal to the mbean deployment URL. Otherwise, it is resolved as a path spec relative to the serverHomeURL (e.g., file:/jboss-3.2.0) using the URL ctor URL(serverHomeURL, codebase) Examples: &lt;classpath codebase='.' archives='util.jar' /&gt; &lt;classpath codebase='http://classloader.dot.com' archives='*' /&gt; &lt;classpath codebase="lib" archives="*"/&gt;  
archives string (default) no (default)     The archives attribute specifies either the '*' wildcard to indicate all contents of the codebase should be included, or a comma seperated list of the jars to include.

Attribute Datatype Required Values Default Comments
code string (default) Yes     The code attributes gives the fully qualified name of the MBean implementation class.  
name string (default) Yes     The name attribute gives the JMX ObjectName string to use when registering the MBean. This must be a unique and valid JMX name.  
xmbean-dd string (default) no (default)     The xmbean-dd attribute defines the path to the JBoss XMBean descriptor. This is an xml document conforming to: &lt;!DOCTYPE mbean PUBLIC "-//JBoss//DTD JBOSS XMBEAN 1.0//EN" "http://www.jboss.org/j2ee/dtd/jboss_xmbean_1_0.dtd"&gt; If the descriptor is embedded via a nested xmbean element, the xmbean-dd attribute must be given as an empty string. Example: &lt;mbean code="org.jboss.test.jmx.xmbean.User" name="jboss.test:service=xmbean-user" xmbean-dd="META-INF/org/jboss/test/jmx/xmbean/User.xml" /&gt;  
xmbean-code string (default) no (default)     The optional xmbean-code attribute specifies the ModelMBean implementation class to use. This defaults to org.jboss.mx.modelmbean.XMBean.
